# 🌪️ Tempest AI
**The Hardware-Aware, Native-Engine Autonomous Engineer.**

Tempest AI is a high-performance, Rust-based autonomous agent designed to be your local "Principal Engineer." Unlike standard chat aliases, Tempest is a **Stateful Intelligence** that operates with a hardened "Frontal Lobe" architecture—enforcing programmatic boundaries, real-time situational awareness, and a disciplined Planning/Execution lifecycle.

---

## 🚀 Recent Upgrades
- **🔌 MCP Protocol Support**: Native integration with the Model Context Protocol. Connect Tempest to any MCP server (Git, SQLite, Jira, Slack) to expand its toolset dynamically.
- **⏪ Multi-Level Undo**: Automatic file snapshots before every modification. Use `/undo` to revert changes across the entire workspace.
- **⚡ Parallel Execution**: High-velocity pipeline that executes independent tool calls in parallel for massive speed gains.
- **🛠️ Slash Commands**: TUI-native commands like `/help`, `/undo`, `/checkpoints`, and `/clear` for seamless session management.

## 🧠 The "Frontal Lobe" Architecture
Tempest has been hardened with a sophisticated agency management layer that prevents hallucinations and unauthorized executions.

### 🛡️ 1. Programmatic Safety Gates & Checkpoints
All system-modifying actions are strictly blocked until an explicit **Implementation Plan** is approved.
- **Automatic Snapshots**: Every file edit is backed up to a `checkpoint.rs` manager before the first byte is written.
- **Visual Diff Previews**: High-fidelity, colorized diffs are generated for all proposed changes during the approval phase.

### 🛰️ 2. State-Injected Turns
Every reasoning turn begins with a high-priority **Situational Report** injected directly into the context:
- **Mode Awareness**: Explicitly informs the model if it is in Planning (Read-only) or Execution mode.
- **Approval Tracking**: Injects the current verification status of the proposed plan.

### 🧪 3. Competency HUD & Tool Stats
The agent monitors its own "Win/Loss" record. 
- **Performance Tracking**: Successful vs. failed tool calls are tallied in a thread-safe registry.
- **Self-Reflective Warnings**: If a tool fails repeatedly, the system injects a **Competency Warning**, forcing the model to stop and analyze the failure.

## 🚀 "Native-Engine" Capabilities

### 🔌 1. Model Context Protocol (MCP) Integration
Tempest is now a first-class MCP client. 
- **Dynamic Discovery**: Connect to any external MCP server via `config.toml`.
- **First-Class Tools**: MCP tools are proxied and registered dynamically, appearing to the LLM exactly like native Rust tools.

### ⚡ 2. Parallel Tool Execution & Normalization
Tempest features a high-velocity execution pipeline.
- **Fan-out Execution**: Independent tool calls (e.g., reading 5 different files) are executed in parallel using `tokio` tasks.
- **DeepSeek Normalizer**: Automatically detects and repacks root-level flat arguments generated by reasoning models into strict schema objects.

### 🧠 3. Persistent Memory & Context Management
- **Context Compaction**: Background summarization is offloaded to a secondary micro-model, preserving the primary reasoning window.
- **Session Resume**: Intelligent session restoration that injects previous progress into the start of a new run.

### 🛠️ Visual Dashboard Features:
- **Real-Time Telemetry**: Live tracking of **CPU, RAM, GPU, and Thermals** in the sidebar.
